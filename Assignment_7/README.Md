## Overview

This assignment extends the **RideShare system from Assignment 5** by adding a statistics analytics layer. Rather than computing statistics on arbitrary numbers, all calculations are performed on real ride data — fares, distances, and ride types — giving each result direct business meaning.

The same analytics are implemented in **three languages**, each representing a different programming paradigm:

| Language | Paradigm | Key concept demonstrated |
|----------|----------|--------------------------|
| **C** | Procedural | Manual memory management, parallel arrays |
| **OCaml** | Functional | Immutable lists, `fold_left`, algebraic types |
| **Python** | Object-Oriented | `RideShareStats` class, encapsulation, `dataclass` |

---

## Statistics Calculated

| Statistic | Business Meaning |
|-----------|-----------------|
| **Mean Fare** | Average amount charged per trip — benchmarks pricing strategy |
| **Median Distance** | Typical trip length — less skewed than mean when long rides exist |
| **Mode Ride Type** | Most commonly booked category — drives fleet allocation decisions |
| **Frequency Table** | Demand split across Standard / Premium / Shared rides |

## How to Run

### Prerequisites

| Language | Requirement | Install |
|----------|-------------|---------|
| C | GCC | MSYS2 UCRT64: `pacman -S mingw-w64-ucrt-x86_64-gcc` |
| OCaml | OCaml interpreter | MSYS2 UCRT64: `pacman -S mingw-w64-ucrt-x86_64-ocaml` |
| Python | Python 3.9+ | https://www.python.org/downloads/ |

> **MSYS2** can be downloaded from https://www.msys2.org  
> Open the **MSYS2 UCRT64** terminal (blue icon) to run `pacman` commands.

---

### Running on Windows

Each folder contains a `.bat` file. **Double-click it** or run from PowerShell:

```powershell
# C
.\C\run_c.bat

# OCaml
.\OCaml\run_ocaml.bat

# Python
.\Python\run_python.bat
```

Each bat file will:
1. Check that the required tool is installed and show a clear message if not
2. Compile (C only) or interpret the source file
3. Print output to the console
4. Save output to `output_<language>.txt` in the same folder

---

### Running Manually

```bash
# C — compile then run
gcc -Wall -o C/stats.exe C/stats.c
./C/stats.exe

# OCaml — interpret directly
# Note: -I flag required for MSYS2 install to find stdlib
C:\msys64\ucrt64\bin\ocaml.exe -I C:\msys64\ucrt64\lib\ocaml OCaml\stats.ml

# Python
python stats.py
```

## Fare Formulas

Fares are calculated using the same pricing rules as Assignment 5:

| Ride Type | Formula |
|-----------|---------|
| Standard | `$2.50 base + $1.20 × distance` |
| Premium | `$5.00 base + $2.80 × distance + $4.00 luxury surcharge` |
| Shared | `($2.00 + $1.00 × distance) ÷ passengers` |

## Known Issue — OCaml on MSYS2

If you see `Error: Unbound module "Stdlib"` when running OCaml, it means the interpreter cannot find its standard library. The `run_ocaml.bat` fixes this automatically by passing:

```bat
ocaml.exe -I C:\msys64\ucrt64\lib\ocaml stats.ml
```

The `-I` flag explicitly tells OCaml where its stdlib lives on Windows. You do **not** need to set any environment variables manually — the bat handles it.
